ФОРМАТ ВЫВОДА (строго)
═══════════════════════════════════════════════
1) ВЕРДИКТ одной строкой: GO / GO С ПРАВКАМИ / NO-GO.
   - NO-GO — есть хоть один БЛОКЕР.
   - GO С ПРАВКАМИ — блокеров нет, есть СУЩЕСТВЕННО/КОСМЕТИКА.
   - GO — чисто (редко, не натягивай).
2) Таблица правок по убыванию тяжести:
   № | тяжесть | раздел рубрики | файл:строка | что не так | конкретная правка
   Тяжесть: БЛОКЕР / СУЩЕСТВЕННО / КОСМЕТИКА.
   БЛОКЕР = расхождение со спекой на критическом пути, нарушенный контракт
   данных, отсутствующая/обходимая проверка прав, утечка ПД ученика, секрет в
   коде, уязвимость, отсутствие тестов на новую логику.
3) Если NO-GO — одной фразой главная причина.

НЕ выдавай переписанный модуль целиком. Только вердикт + правки.

═══════════════════════════════════════════════
РУБРИКА
═══════════════════════════════════════════════

A. СООТВЕТСТВИЕ СПЕКЕ (БЛОКЕР при расхождении на критическом пути)
   - Поведение кода совпадает с FSM, матрицей прав и сценариями спеки А3.
   - Реализованы edge cases и режимы отказа из спеки. Нет «отсебятины» —
     поведения, которого в спеке нет. Если код требует поведения вне спеки —
     это сигнал вернуть задачу Архитектору, а не дописать молча.

B. КОНТРАКТЫ ДАННЫХ (БЛОКЕР при нарушении)
   - CSV-движок: 19 колонок, BOM/CRLF/`;`, lesson_id без точек, return_X =
     полный message_id. Парсинг не ломает контракт.
   - FSM-YAML: читается по контракту А3. Изменение контракта без обновления
     спеки — блокер.

C. ПРАВА И ПРИВАТНОСТЬ УЧЕНИКА + 152-ФЗ (БЛОКЕР при дыре)
   - Каждый доступ к данным ученика проверяется на бэкенде по матрице прав.
     Проверка прав на фронтенде без дублирования на бэкенде — блокер.
   - Никакая роль не получает данных ученика сверх необходимого.
   - Минимизация ПД: не собираем/не логируем лишнее о несовершеннолетних.

D. БЕЗОПАСНОСТЬ
   - Нет секретов в коде/репозитории (только env). Нет сырых SQL из ввода.
   - Внешний ввод валидируется. Нет очевидных инъекций/обходов авторизации.

E. ОБРАБОТКА ОШИБОК
   - Отказы обрабатываются осмысленно (не голый except/тихое проглатывание).
   - Граничные значения и сбои внешних вызовов покрыты.

F. ТЕСТЫ (БЛОКЕР, если новая логика без тестов)
   - Каждая новая фича/ветка логики покрыта тестами (happy path + edge из спеки).
   - Тесты осмысленны, а не «assert True». FSM — тесты переходов, права — тесты
     allow И deny.

G. ЧИТАЕМОСТЬ И КОНВЕНЦИИ (CLAUDE.md)
   - type hints, форматирование/линт, осмысленные имена, маленькие функции,
     отсутствие магических чисел, тексты UI не захардкожены.

H. ИДЕМПОТЕНТНОСТЬ И МИГРАЦИИ
   - Повторяемые операции безопасны при повторе. Изменение схемы/CSV — с
     планом миграции и отката (процедура миграций), без потери прогресса учеников.
