ФОРМАТ ВЫВОДА (строго)
═══════════════════════════════════════════════
1) ВЕРДИКТ одной строкой: GO / GO С ПРАВКАМИ / NO-GO.
   - NO-GO — если есть хоть один БЛОКЕР.
   - GO С ПРАВКАМИ — блокеров нет, но есть СУЩЕСТВЕННО/КОСМЕТИКА.
   - GO — чисто (редко, не натягивай).
2) Таблица правок, по убыванию тяжести. Каждая строка:
   № | тяжесть | раздел Методологии | где (message_id) | что не так | конкретная правка
   Тяжесть: БЛОКЕР / СУЩЕСТВЕННО / КОСМЕТИКА.
   БЛОКЕР = нарушение конституции, неверная арифметика, неверный правильный ответ,
   выход за рамки ЦА/программы, аббревиатура метода, тон-нарушение в негативе.
3) Блок «АРИФМЕТИКА» — твой независимый пересчёт (см. ниже), вывод Python.
4) Блок «ФИПИ» — результат сверки с 3 PDF (см. модуль ниже).
5) Если NO-GO — одной фразой главная причина.

НЕ выдавай переписанный урок. Только вердикт + правки.

РУБРИКА (по разделам Methodology v2.1)
═══════════════════════════════════════════════

A. ЗАТУХАНИЕ МЕТОДА ЧТЕНИЯ — раздел 3.2.1 (СЕЙЧАС ГЛАВНЫЙ РИСК)
   - Определи позицию урока: 1.1 источник · 1.2 +1 · 1.3 +2 · 1.4 +3 ·
     1.5–1.8 фон · 1.9 явный возврат.
   - Степень явности метода (что дано → что спрашивают → как связаны → проверка)
     ДОЛЖНА совпадать с позицией (таблица затухания 3.2.1):
     · источник — учим явно, 4 шага = главное содержание;
     · +1 — полностью в образце, перечисляем по смыслу;
     · +2 — видимо в образце, без ритуала;
     · +3 — одна естественная фраза, без перечисления шагов;
     · фон — в обучающих сообщениях НЕ упоминаем (можно 1 раз в фидбэке);
     · возврат (мета) — явно показываем, что метод работал на всех сюжетах.
   - БЛОКЕР, если в фоновом уроке (1.5–1.8) метод проговаривается явно по шагам —
     это превращает его в ритуал.
   - БЛОКЕР, если есть ЛЮБАЯ аббревиатура/кодовое имя метода (в т.ч. по первым
     буквам шагов). Шаг называется смыслом — «что дано», не буквой. ЧЧГП удалена.

B. АРИФМЕТИКА И ВЕРНЫЕ ОТВЕТЫ — раздел 3.5 (БЛОКЕР при любой ошибке)
   - Пересчитай КАЖДОЕ число сам, Python-скриптом в bash_tool. Не верь числам урока.
   - Для каждого вопроса Q1–Q4, Q4b, R1–R3: реши задачу с нуля, получи верный ответ,
     сверь с correct_answer в CSV.
   - Каждый НЕВЕРНЫЙ вариант обязан быть результатом КОНКРЕТНОЙ типичной ошибки
     (забыл перевести единицы, спутал площадь/периметр, не округлил…), а не случайным
     числом. Если не можешь назвать ошибку за вариантом — это дефект (СУЩЕСТВЕННО).
   - Единицы консистентны (м/мм, ₽/коп., %/доли).
   - Numeric collision: значение правильного ответа не совпадает ни с одним числом
     из условия задачи. Совпадение — СУЩЕСТВЕННО (угадывание по совпадению).

C. БУКВА ОТВЕТА — сверка по CSV + независимый пересчёт (БЛОКЕР)
   - docx больше нет. correct_answer (англ. A/B/C/D) сверяется с буквой того варианта
     (option_a..d), который соответствует твоему независимому верному ответу из
     раздела B. Если correct_answer указывает не на тот вариант — БЛОКЕР.
   - Соответствие латиница↔вариант: A=option_a, B=option_b, C=option_c, D=option_d.
   - ОСОБО: проверь, что буква не скопирована машинально с соседнего урока.
     Известные ловушки блока 1: у урока 1.4 главный вопрос = A (у соседей B);
     у урока 1.9 = B (Q4 и Q4b). Если буква «подозрительно как у всех» — перепроверь руками.
   - Распределение букв Q1→Q4 не должно давать 4 одинаковых подряд (инерция угадывания).

D. СТАНДАРТ ВОПРОСОВ — раздел 3.3
   - Q1 узнавание (~20%) · Q2 применение (~50%) · Q3 перенос или ловушка (~20–30%).
   - 3–4 варианта, однозначно различимы, вопрос понимаем без двусмысленности.
   - Типы дистракторов в одном уроке концептуально различны (не два «потерял разряд»).

E. ОБРАТНАЯ СВЯЗЬ НА ОШИБКУ — раздел 3.4 (тон-нарушения = БЛОКЕР)
   - НЕТ слов «неправильно/неверно» в негативе, НЕТ восклицательных знаков пачками,
     НЕТ менторских интонаций («как известно», «запомните»).
   - На каждый неверный вариант — объяснение 1–3 строки + return_X к КЛЮЧЕВОМУ
     обучающему сообщению (теории с алгоритмом), а не к началу урока.
   - return_X = полный message_id целевого сообщения (с префиксом для 1.2+).
     Указание на несуществующий message_id — БЛОКЕР (keeper.py тоже это ловит).

F. ТОН И ЯЗЫК — разделы 3.1 + антипринципы 1.3
   - «Ты», не «вы». Тон ровесника-«старшего товарища».
   - Нет наказания за ошибку, нет дедлайнов/сгорания доступа, нет сравнения с другими.
   - Хук не пустой (объясняет «зачем»). Финал без приторности.

G. МИКРО-ТЕОРИЯ И СТРУКТУРА — раздел 3.2 + архетип блока
   - Ориентир 1,5 экрана (~14 строк). НО смысл важнее разбивки: цельный список/
     цепочку/формулу ≤6 строк НЕ дробим ради счётчика. Режем только по смысловой границе.
   - Заголовок «что узнаем», якорь из жизни, 2–3 шага логики, фраза-вывод.
   - СТРУКТУРУ урока (число образцов/вопросов/сообщений теории, наличие чертежа)
     судим по архетипу блока: блок 1 — референс урок 1.1; блоки 2+ — структурный
     бриф блока (дефолт 2.1). Каркас из 6 этапов (хук→теория→образец→тренировка→
     главный→финал) — инвариант. Отклонение наполнения, обоснованное эффективностью
     и сохраняющее инварианты (15 мин, recall, mastery, повторения, тон), — НЕ дефект.
     Слепое копирование педагогической формы блока 1 в другой блок — дефект (СУЩЕСТВЕННО).

H. АЛГОРИТМ vs ФОРМУЛА — раздел 3.2
   - По умолчанию словесный алгоритм. Формула — только если: алгоритм длиннее в 1,5+ раза,
     ИЛИ тема — собственно формулы (блоки 2–3), ИЛИ ФИПИ требует формульную запись.
   - Если в уроке формула, а можно алгоритмом — СУЩЕСТВЕННО.

I. MASTERY И ПОВТОРЕНИЯ — разделы 2.2.1 + 2.2
   - Есть Q4 (main_question) + Q4b (main_question_backup) + lesson_failed (спокойный тон).
   - R1 (через 1ч) / R2 (вечер) / R3 (утро ☀️) работают на ТОТ ЖЕ навык, что Q4,
     а не на побочный. Каждый — другой контекст.

J. РАМКИ ПРОГРАММЫ — Project Brain 2.4 + Methodology 9.3 (выход = БЛОКЕР)
   - Урок НЕ опирается на зад. 13, 14, 16, 17, теорему Виета, часть 2.
   - Сложность — уровень «тройки», не выше. ЦА — слабый ученик с арифметикой 5–6 класса.

K. ФОРМАТ (semantic spot-check; байты проверяет keeper.py отдельно)
   - <b> для терминов/шагов/итоговых чисел; <i> для цитат условия; <code> для
     чисел/формул/размеров/маркировок. Эмодзи ≤2 на сообщение, из whitelist.
   - Глубокую проверку байтов (BOM/CRLF/19 колонок/парность тегов/ссылки return_X)
     НЕ делай руками — это работа Хранителя (keeper.py). Отметь только явные
     смысловые сбои разметки.
