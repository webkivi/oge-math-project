**Версия:** v1
**Источники (приоритет):** Methodology v2.1 > Project Brain v3.2 > А3: `student_lesson_fsm_v4` + `student_registration_fsm_v2`

**Вне охвата:** родитель / учитель / репетитор (их FSM ещё не написаны), привязка взрослых по коду, мессенджер-поверхности. Эти компоненты появятся, когда выйдут соответствующие спеки А3.

## 0. Рамки и честное ограничение

То, что ниже, — **компетентные дефолты, а не выверенный UX.** Настоящий UX-цикл — наблюдение за 5+ живыми 9-классниками на пилоте. «Спроектировано А6» ≠ «проверено на пользователях». Каждый экран помечен привязкой к состоянию/событию FSM, чтобы режим разработки и А4 могли проверить соответствие.

**Дизайн-направление (обоснование одной фразой).** Тон Методологии — «старший товарищ, который просто шарит»: спокойно, по-взрослому, без инфантильности и давления. Продукт — место отдыха от экзаменационного стресса. Поэтому единственная намеренная «смелость» макета — **спокойная, ненаказывающая подача ошибки** (тёплая охра «типичная ловушка» вместо тревожного красного — прямое следствие антипринципа §1.3 «не наказываем за ошибки»). Всё остальное держим тихим и дисциплинированным: низкая визуальная возбудимость, высокая читаемость, минимальная длина пути.

**Развилки фаундеру (вынесены, дизайном НЕ закрыты).** Их решает фаундер (вход А9):

| # | Развилка | Что за ней | Дефолт, который я заложил под тест |
|---|----------|-----------|------------------------------------|

**Развилки, уже закрытые/трактованные А3** (мне их закрывать не нужно, но они касаются UI):
- **D-3** (передышка) — А3 принял «автоматически». Значит **кнопки «использовать передышку» нет**; есть только спокойное информирование постфактум (S-07).
- **D-4** (разминка) — push в 08:00, но разминка доступна при любом входе. UI: вход в разминку с `daily_start`, не привязан жёстко ко времени.
- **D-5** (офлайн в уроке) — заглушка «нет соединения» без потери прогресса. UI: состояние `ConnectionStub` (§5).
- **D-6** (grade=8) — закрыта жёстким гейтом в production. UI: класс «8» в production не выбирается; экран `gate_grade8` — тёплый, не наказывающий.
- **Z-2** (текст политики ПД) — зона юриста/А8. Я ставлю **ссылку + обязательный чекбокс**, текст не пишу.

**Противоречий Методологии, блокирующих задачу, не найдено.** Счётчик дней — это «вознаграждение» из петли привычки §1.1, а не запрещённый «давящий счётчик», пока подан тихо; одно блокирующее окно (grade=8) — юридическое (152-ФЗ), а не удержание. Это вопросы тона и заметности, решаемые дизайном, а не блокеры.

## 1. Дизайн-токены

Именованные токены, готовые лечь в `tailwind.config.ts` (раздел `theme.extend`). Контраст текста проверен по WCAG 2.1 AA (нормальный текст ≥ 4.5:1; UI-границы/крупный текст ≥ 3:1).

### 1.2 Типографика

**Шрифт — системный гуманистический санс-стек, без веб-шрифта.** Обоснование под PWA-офлайн: нет загрузки веб-шрифта на нестабильном соединении (нет FOUT/мигания), полное покрытие кириллицы из коробки, мгновенный рендер из кэша service worker.

```
--font-sans: ui-sans-serif, system-ui, -apple-system, "Segoe UI",
             Roboto, "Helvetica Neue", Arial, sans-serif;
--font-mono: ui-monospace, "SF Mono", "Cascadia Mono", Menlo, monospace; /* числа/единицы в условии */
```

| Роль | Токен | Размер / межстрочный | Вес |
|------|-------|----------------------|-----|
| Заголовок экрана | `text-title` | 24px / 1.3 | 700 |
| Подзаголовок / хук | `text-lead` | 19px / 1.4 | 600 |
| Основной текст урока | `text-body` | 17px / 1.6 | 400 |
| Вариант ответа | `text-option` | 17px / 1.4 | 500 |
| Подпись / счётчик / метка | `text-caption` | 14px / 1.4 | 500 |

Базовый кегль **17px** (не 16) — комфорт чтения с телефона для уставшего подростка; межстрочный 1.6 в теле — против «стены текста». «Правило 1,5 экранов» (§3.2 Методологии) реализуется длиной контента, не размером шрифта.

### 1.3 Отступы, сетка, размеры

- База — **4px**. Шкала: `1=4 · 2=8 · 3=12 · 4=16 · 5=20 · 6=24 · 8=32 · 10=40 · 12=48`.
- **Mobile-first, одна колонка.** Контентная ширина `max-w-[440px]`, центр, боковые поля `16px`. На широком экране контент не растягивается — урок остаётся «телефонным».
- Вертикальный ритм между блоками сообщения — `16–24px`. Между сообщениями урока — пауза-разделитель `24px` (визуальный аналог «задержки на стрелке» из §3.7).
- Нижняя «зона действия» (кнопка «Дальше» / варианты) закреплена с отступом `16px` от низа, в безопасной зоне (`env(safe-area-inset-bottom)`).

### 1.4 Радиусы

| Токен | Значение | Где |
|-------|----------|-----|
| `radius-card` | 16px | карточки урока, сообщения, диалоги |
| `radius-control` | 12px | кнопки, поля, варианты ответа |
| `radius-pill` | 999px | счётчик «дни подряд», метки блока |

### 1.5 Состояния (hover / active / disabled / focus)

| Состояние | Правило | Токены |
|-----------|---------|--------|
| **hover** (только указатель) | главное действие → `--primary-hover`; карточки → `--bg-sunken` подложка | `--primary-hover` |
| **active / pressed** | главное действие → `--primary-active`; лёгкое «утопление» `translate-y-[1px]` | `--primary-active` |
| **disabled** | `--disabled-bg` + `--disabled-text`, `cursor: not-allowed`, без hover-реакции; кнопка «Дальше» неактивна, пока guard не выполнен (имя пустое, чекбокс не стоит) | `--disabled-bg`, `--disabled-text` |
| **focus-visible** | видимое кольцо `2px` `--focus-ring` + `2px` offset; **никогда не убираем** outline без видимой замены | `--focus-ring` |
| **selected** (вариант/класс) | граница `2px` `--primary` + подложка `--correct-bg`/нейтральная; не только цвет — добавить галочку/точку (не полагаемся на цвет) | `--primary` |

Все состояния-через-цвет дублируются нецветовым признаком (иконка/толщина/текст) — требование доступности (не полагаться только на цвет).

## 3. Экраны и потоки — роль «Ученик»

Уровень — wireframe (что на экране и зачем), не пиксельный макет. **Каждый экран привязан к состоянию FSM; каждый переход — событие.** Состояние `streak_update` — без UI (синхронный серверный переход, ученик не взаимодействует; §2 lesson_v4).

### 3.1 Поток регистрации (`student_registration_fsm_v2`)

Минимальный путь для grade=9: **3 экрана** (имя → класс → согласие). Для grade 10/11 — +1 уточнение. Лишних экранов нет.

```
unregistered ─evt_open_pwa─▶ name_entry ─evt_name_submitted─▶ grade_entry ─┬─[8/prod]──evt_grade_selected─▶ gate_grade8
                                                                            ├─[9]──────evt_grade_selected─▶ consent_gate
                                                                            └─[10,11]──evt_grade_selected─▶ ogeprep_check
ogeprep_check ─evt_ogeprep_yes─▶ consent_gate    ogeprep_check ─evt_ogeprep_no─▶ course_mismatch ─evt_mismatch_continue─▶ consent_gate
consent_gate ─evt_submit_registration─▶ registered  (→ v4: daily_start → первый урок)
любой шаг ─evt_cancel_registration─▶ unregistered     назад ─evt_back─▶ предыдущий шаг
```

**`name_entry`** — состояние `name_entry`
```
┌──────────────────────────────┐
│  ● ○ ○   (StepDots)           │
│  [A8: заголовок «как тебя     │
│        зовут?» — ник, не ФИО] │
│  ┌────────────────────────┐  │
│  │ TextField (никнейм)    │  │
│  └────────────────────────┘  │
│  [A8: подсказка «настоящее    │
│   имя не нужно»]              │
│            [ Дальше ]  ←disabled пока пусто (RC-02)
└──────────────────────────────┘
```
Переходы: `Дальше`→`evt_name_submitted`→`grade_entry`; закрыть→`evt_cancel_registration`. Назад из первого шага = отмена (по FSM `evt_back` из `name_entry` невозможен).

**`grade_entry`** — состояние `grade_entry`. `ChoiceCard` ×3 (9/10/11; в staging +«8» с `InfoBanner`-предупреждением и явным подтверждением; в production «8» отсутствует — RC-04/RC-05 страхует бэкенд).
Переходы: выбор→`evt_grade_selected`→{`gate_grade8` | `consent_gate`(9) | `ogeprep_check`(10/11)}; `Назад`→`evt_back`→`name_entry` (имя сохранено в draft).

**`ogeprep_check`** (только 10/11) — состояние `ogeprep_check`. Один вопрос: готовишься/пересдаёшь ОГЭ? Два `ChoiceCard`: да / нет.
Переходы: да→`evt_ogeprep_yes`→`consent_gate`; нет→`evt_ogeprep_no`→`course_mismatch`; `Назад`→`evt_back`→`grade_entry`.

**`course_mismatch`** (только из `ogeprep_check`) — **информирует, не блокирует** (§1.4). Текст «курс заточен под тройку по ОГЭ»; две равноправные кнопки.
Переходы: `Всё равно начать`→`evt_mismatch_continue`→`consent_gate`; `Выйти`→`evt_mismatch_leave`→`unregistered` (уход по своему выбору, без удержания — RC-12).

**`consent_gate`** — состояние `consent_gate`. Опциональный `InfoBanner` самооценки для grade=9 (целюсь на 4–5 / за 2 недели / не освоил арифметику — **UI-информер вне FSM**, не блокирует). `PolicyLink` + обязательный `Checkbox` + `Button` «Начать».
```
┌──────────────────────────────┐
│  ○ ○ ●                        │
│  [InfoBanner grade=9 — опц.]   │
│  [A8: «прежде чем начать»]     │
│  [ PolicyLink: Политика ПД ]   │
│  ☐ [A8: согласие на обработку ПД]  ←обязателен (RC-03)
│            [ Начать ]  ←disabled пока ☐ пуст / политика не загрузилась (RF-04)
└──────────────────────────────┘
```
Переходы: `Начать`→`evt_submit_registration` [guard: имя≠пусто AND grade∈9..11 AND consent=true; идемпотентность по `onboarding_session_id`]→`registered`. `Назад`→`evt_back`→`grade_entry` (согласие сбрасывается, `policy_version_shown` перечитывается при возврате — RC-14). Кнопка блокируется после первого нажатия (RC-01).

**`gate_grade8`** — состояние `gate_grade8`. Тёплый, не наказывающий экран; аккаунт не создаётся.
Переход: `Понятно`→`evt_gate_dismiss`→`unregistered` (draft уничтожен).

**`registered` (стык)** — конец под-FSM регистрации = `registered` в v4. Передача в `daily_start`→первый урок. Если первый урок не подгрузился (RF-06) — показываем `ConnectionStub`, аккаунт уже создан, прогресс не теряется.

### 3.2 Дневной поток и урок (`student_lesson_fsm_v4`)

```
registered ─evt_open_app─▶ daily_start ─┬─evt_warmup_available─▶ morning_warmup ─evt_warmup_complete─▶ lesson_select
                                         └─evt_warmup_skip──────────────────────────────────────────▶ lesson_select
lesson_select ─evt_start_lesson─▶ lesson_hook ─▶ lesson_theory ─▶ lesson_example ─▶ lesson_training ⟲
lesson_training ─(все Q1..Q3)─▶ lesson_main_question ─correct#1─▶ lesson_final
lesson_main_question ─wrong#1─▶ lesson_theory_review ─▶ lesson_main_question_backup ─correct#2─▶ lesson_final
lesson_main_question_backup ─wrong#2─▶ lesson_failed ─evt_lesson_fail_confirmed─▶ daily_blocked
lesson_final ─evt_lesson_complete─▶ repeat_1h_pending ⟲ ─evt_1h_elapsed─▶ repeat_1h_active
   ─▶ repeat_evening_pending ⟲ ─evt_evening_time─▶ repeat_evening_active ─▶ review_queue_scheduled ─▶ registered
lesson_select ─evt_no_lesson_today─▶ {daily_done | daily_blocked}   lesson_select ─evt_all_lessons_done─▶ course_complete
В ЛЮБОМ состоянии урока: «Выйти из урока» ─evt_cancel_lesson─▶ registered (прогресс сохранён, урок возобновляем)
```

**`daily_start`** — состояние `daily_start`. Хаб дня, **одна мысль: «твой сегодняшний шаг»**. Один главный CTA. Тихий `DayCounterBadge` (**F-1**). Если есть due-повторения — кнопка ведёт в разминку; иначе — сразу к уроку.
```
┌──────────────────────────────┐
│  [DayCounterBadge: N дней]  ← тихо, вторично (F-1)
│  [A8: приветствие дня]        │
│  ┌── главный шаг дня ──────┐  │
│  │ [A8: «начать разминку» │  │  ← evt_warmup_available
│  │  или «открыть урок»]   │  │  ← evt_warmup_skip (queue пуста)
│  └────────────────────────┘  │
│  [ Карта знаний ] [ Настройки ]
└──────────────────────────────┘
```
Переходы: `evt_warmup_available`→`morning_warmup`; `evt_warmup_skip`→`lesson_select` (queue пуста ИЛИ явный «Пропустить разминку»).

**`morning_warmup`** — состояние `morning_warmup`. `WarmupRunner`: 3 коротких interleaved-вопроса. Кнопка `Пропустить разминку` (явный `evt_warmup_skip`).
Переход: `evt_warmup_complete`→`lesson_select`.

**`lesson_select`** — транзитное состояние (выбор следующего урока). Обычно без отдельного экрана: если урок есть — сразу `lesson_hook`. Имеет смысл показать только три тупиковых исхода: `daily_done`, `daily_blocked`, `course_complete`.

**Прохождение урока** (`lesson_hook → theory → example → training → main_question → [theory_review → main_question_backup] → final`). Каждое под-состояние — экран на базе `MessageCard`/`QuestionBlock`. Шапка несёт `LessonProgress` (этап 1–6) и кнопку **«Выйти из урока»** (на каждом — `evt_cancel_lesson`→`registered`, прогресс сохранён).
```
┌──────────────────────────────┐
│ ← Выйти из урока   [▮▮▯▯▯▯]   │  ← evt_cancel_lesson / LessonProgress
│  ┌── MessageCard ─────────┐  │
│  │ [A8: содержимое этапа] │  │  hook/theory/example/final = MessageCard
│  └────────────────────────┘  │
│  (training/main = QuestionBlock + AnswerButtons A–D)
│            [ Дальше ]         │  ← evt_hook_read / evt_theory_read / evt_example_read
└──────────────────────────────┘
```
- `lesson_training` (⟲): верный вариант → `evt_answer_correct` (остались вопросы — петля; все Q1..Q3 пройдены — движок сам переходит в `lesson_main_question`, без отдельного события — §8 контракт v4). Неверный → `evt_answer_wrong`: `AnswerFeedback` в тоне «типичная ловушка» (`--trap`) + возврат к нужному куску теории по `return_X`. 3 ошибки подряд на одном вопросе → `evt_training_max_errors`→`lesson_failed`.
- `lesson_main_question`: верно с 1-й → `evt_main_correct_attempt1`→`lesson_final`. Неверно → `evt_main_wrong_attempt1`→`lesson_theory_review` (спокойный повтор куска теории) → `evt_theory_reviewed`→`lesson_main_question_backup` (другой пример). Верно со 2-й → `evt_main_correct_attempt2`→`lesson_final`; неверно → `evt_main_wrong_attempt2`→`lesson_failed`.
- `lesson_final`: подбадривание без приторности, `+1` к счётчику, обновление карты знаний. `evt_lesson_complete`→`repeat_1h_pending` (или `course_complete`, если пройдены все 27).

**`lesson_failed`** — состояние `lesson_failed`. `DeferCard`: «отложим до завтра», **без вины, без наказания** (§1.3). Прогресс: `status=failed_today`, урок в `review_queue` на завтра.
Переход: `Понятно`→`evt_lesson_fail_confirmed`→`daily_blocked`.

**`daily_blocked`** (⟲) — состояние `daily_blocked`. `PendingNotice`: «новый урок откроется завтра». Самопетля при повторном входе (`evt_open_app`). Сегодня можно делать только разминку/повторения.
Переход: `evt_next_day`→`registered`.

**Повторения** — `repeat_1h_pending`(⟲: «R1 будет через N минут») → `repeat_1h_active` (1 короткий вопрос) → `repeat_evening_pending`(⟲) → `repeat_evening_active`(⟲: «вечернее повторение ждёт») → `review_queue_scheduled` → `registered`. Все `*_pending` — это `PendingNotice` (информирование, не действие), все `*_active` — `QuestionBlock`. Пропуск интервала откатывает его назад (`evt_next_day`), без наказания.

**`daily_done`** — состояние `daily_done`. `EmptyState`: «на сегодня всё, отдыхай». Спокойно, без догоняющих предложений «ещё урок».
Переход: `evt_session_end`/`evt_next_day`→`registered`.

**`course_complete`** — состояние `course_complete`. Поздравление + доступ к карте знаний и настройкам. Не терминал: `evt_open_app`→`registered`, `evt_delete_account`→`unregistered`.

**Настройки и удаление аккаунта** — экран `AccountSettings` (доступен из `daily_start`). Сейчас — только просмотр профиля (ник, класс) + **«Удалить аккаунт»**. Редактирования имени/класса **нет** (D-1 = deny; **F-2**: если фаундер откроет — добавляется экран «Изменить имя/класс»).
Удаление: `ConfirmDialog` «все данные удалятся безвозвратно» → `evt_delete_account` → каскадное удаление → `Toast`/экран «Аккаунт удалён» → `unregistered`.

## 5. Доступность и PWA-нюансы

**Тач и клавиатура.** Все интерактивные цели ≥ **44×44px** (кнопки `min-h-48px`). Полная навигация с клавиатуры; видимый `focus-visible` (`--focus-ring`, 2px + offset), логичный порядок таба. На смене экрана и при открытии `ConfirmDialog` — программный перевод фокуса на заголовок/первый контрол; в диалоге — фокус-ловушка, `Esc` закрывает (= отмена). Состояния-через-цвет дублируются иконкой/текстом (не только цвет).

**Движение.** `prefers-reduced-motion` уважается: паузы между сообщениями и переходы упрощаются до мгновенных, без параллакса и «фейерверков» (и так не используем — §1.3).

**Офлайн (service worker).**
- Оболочка онбординга (`name_entry`/`grade_entry`) кэшируется и работает офлайн; `evt_submit_registration` откладывается до сети, ПД офлайн не уходят (RF-03). При сбое сабмита — `[A8: «не удалось, попробуй ещё»]`, draft в памяти, повтор идемпотентен по `onboarding_session_id` (RF-01).
- Контент урока, прочитанный/закэшированный, доступен офлайн. Если контент не в кэше (D-5) — `ConnectionStub` «нет соединения» **без потери прогресса**; возобновление при сети.
- `pd_policy` не загрузилась → чекбокс согласия неактивен, регистрацию нельзя завершить (RF-04, информированность согласия 152-ФЗ).

**Установка «на экран Домой».** PWA ставится без магазина. На iPhone — двухшаговый онбординг: (1) «Добавить на экран Домой» (иконка), (2) **отдельно** разрешение на push. Push **не** запрашиваем на первом экране и **не** при регистрации (`pwa_push_token=null` при регистрации; `push_token_own/create` — только при явном согласии). Дефолт: предлагаем push **после первого успешного урока**, спокойно, с возможностью отказаться (отзыв = `push_token_own/delete`). *Момент запроса push соприкасается с тоном — оставляю компетентным дефолтом под пилот.*

**Пустые и ошибочные состояния** (как UI-состояния, не «настроение»): `daily_done`/нет уроков → `EmptyState`-приглашение; `gate_grade8` → тёплый гейт; офлайн → `ConnectionStub`; сбой регистрации (RF-01/RF-02) → ученик остаётся в `consent_gate` с понятным сообщением, без «висячего» аккаунта; первый урок не подгрузился (RF-06) → `ConnectionStub`, аккаунт цел.
