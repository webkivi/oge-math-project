# КАК ЗАПУСТИТЬ ПРОЕКТ

Это полная папка проекта. Всё уже разложено по местам.

## Шаг 1. Распакуй этот архив

Создай на компьютере папку, например:
  C:\Проекты\oge-math

Распакуй архив туда. Внутри должны оказаться файлы и папки прямо в oge-math,
а не вложенная ещё одна папка. То есть так:

  oge-math\CLAUDE.md              ← этот файл видно сразу
  oge-math\Methodology_v2_1.md
  oge-math\.claude\               ← папка с точкой в начале (это нормально)
  oge-math\content\               ← уроки
  ...и т.д.

Если после распаковки внутри oge-math лежит ещё одна папка project-ready —
перетащи всё из неё на уровень выше, а пустую project-ready удали.

Важно: папка .claude начинается с точки. Windows может её прятать.
Если не видишь — в проводнике нажми «Вид → Показать скрытые элементы».

## Шаг 2. Открой в Claude Code

Открой приложение Claude Code. Нажми «Open folder» (или «Открыть папку»)
и выбери oge-math. Claude Code прочитает CLAUDE.md и поймёт, что это за проект.

## Шаг 3. Проверь, что всё работает

Напиши в окне обычными словами:

  Прочитай CLAUDE.md и расскажи, как устроен проект.

Если Claude ответил по делу (упомянул ОГЭ, PWA, 4 роли) — всё работает.

## Шаг 4. Начинай работу

Первая настоящая задача по плану:

  /фундамент

Это команда, которая запустит проектирование ядра: как ученик проходит урок.
Claude сам вызовет нужных «агентов», проверит результат и покажет тебе итог.

## Что внутри (не надо открывать и менять руками)

  .claude/          — инструкции для Claude Code (агенты, команды, проверки)
  tools/            — скрипт-проверщик validator.py
  prompts/          — тексты ролей агентов (уже синхронизированы под v2.1/v3.2)
  content/          — уроки 1.1–1.9 в CSV
  fipi/             — документы ФИПИ (кодификатор, спецификация, демоверсия)
  specs/            — сюда Claude будет класть архитектурные спецификации
  CLAUDE.md         — главные правила для Claude Code
  Methodology_v2_1  — конституция продукта
  Project_Brain_v3_2 — стратегия и статус
