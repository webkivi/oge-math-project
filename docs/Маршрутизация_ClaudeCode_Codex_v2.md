# Маршрутизация задач: Claude Code vs Codex

Версия: v2
Дата: 2026-06-27
Проект: oge-math-project

## Быстрое правило

**Новое поведение / архитектура / контракт → Claude Code**
**Ревью / проверка / sync / узкая реализация → Codex**

---

## Claude Code — берёшь сюда если:

- принятой спеки ещё нет
- поведение системы меняется
- нужен А3 Архитектор или А4 Критик системы
- может измениться контракт данных (CSV / FSM / API)
- затронуты права, ПД, приватность
- есть продуктовая развилка
- меняются несколько подсистем сразу
- нужно обновить Brain / Methodology / CLAUDE.md

## Codex — берёшь сюда если:

- спека уже принята
- нужен ревью дифа (по рубрике А5)
- нужно проверить состояние репозитория
- нужно сделать git push / fetch / merge / compare
- нужно синхронизировать GitLab
- нужен узкий багфикс в рамках принятой спеки
- нужна независимая инженерная проверка

---

## Привязка к ролям проекта

| Роль | Инструмент |
|---|---|
| А3 Архитектор | Claude Code |
| А4 Критик системы | Claude Code |
| А5 Ревьюер кода | Codex (читает prompts/Агент_5_Ревьюер_кода_v1.txt) |
| А8 Копирайтер | Claude Code (субагент) |
| А9 Продуктолог | Claude Code (субагент) |
| git push/fetch/merge | Codex |
| GitLab sync | Codex |
| Low-risk fix | Codex |
| Brain / Methodology | Claude Code |

## Развилка по уровню риска

```
High-risk → Claude Code (всегда)

Mid-risk → архитектурное суждение ещё активно? → Claude Code
         → спека принята, работа узкая?         → Codex

Low-risk → Codex (по умолчанию)
```

## Правило сомнения

Не уверен куда?
- Вопрос про «как должно работать» → Claude Code
- Вопрос про «как сейчас работает / проверить / закоммитить» → Codex
