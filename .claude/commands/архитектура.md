Запусти архитектурный конвейер для задачи: **$ARGUMENTS**.
Уровень риска по умолчанию High (новая модель/роль/права) — полный конвейер.

Шаги:

1. **Архитектор.** Делегируй субагенту `architect`: задача «$ARGUMENTS».
   Он читает Methodology/Brain + существующие `specs/` и пишет спеку
   `specs/<задача>.md` (FSM-YAML + матрица прав + сценарии + edge cases).

2. **validator.py (шлюз).** Прогони `python tools/validator.py specs/<задача>.md`.
   Если FAIL — верни нарушения субагенту `architect` на исправление, повтори,
   пока validator не даст PASS.

3. **Критик системы — В ИЗОЛЯЦИИ.** Делегируй субагенту `system-critic`. Передай
   ТОЛЬКО: путь к спеке + указание читать Methodology/Brain + ранее принятые
   `specs/`. НЕ передавай рассуждения Архитектора и историю итераций.

4. **Цикл.** Вердикт GO WITH REVISIONS → передай правки субагенту `architect`,
   снова validator (шаг 2), снова критик изолированно (шаг 3). До вердикта GO.
   NO-GO → остановись, покажи причину фаундеру.

5. **Итог фаундеру.** При GO + validator PASS покажи путь к спеке, краткую сводку
   и `git diff`. НЕ коммить сам — спека это контракт, её принимает фаундер.

Если в задаче всплыла нерешённая продуктовая развилка — не угадывай. Останови
конвейер, назови развилку и предложи прогнать её через субагента
`product-strategist` (А9), затем дождись решения фаундера.
