Заложи фундамент платформы. Это первая реальная работа по коду: цель —
формально описать ядро (ученик проходит урок), а не «писать бота наугад».

Последовательность строго:

1. Сначала убедись, что среда готова: `git status` (репо инициализирован),
   `CLAUDE.md` в корне, промпты в `prompts/`, validator на месте
   (`python tools/validator.py --help`). Чего нет — скажи фаундеру, не обходи.

2. Запусти архитектурный конвейер (как `/архитектура`) для задачи
   **«логика роли ученик и прохождение урока в PWA»**. Эта задача также фиксирует
   раскладку каталогов репозитория — зафиксируй её в спеке. Прогони через
   architect → validator.py → system-critic до GO.

3. Когда спека ученика принята (GO + validator PASS) — остановись и покажи
   фаундеру итог. НЕ переходи к написанию бэкенд-кода без его «ок»: следующий
   шаг (модель данных + движок CSV + FSM-движок урока) — это уже High-risk код
   по принятой спеке, он идёт отдельной сессией в режиме разработки по CLAUDE.md.

Не расширяй охват на родителя/учителя/репетитора и мессенджеры — сначала ученик
end-to-end. Это решение фаундера по критическому пути.
